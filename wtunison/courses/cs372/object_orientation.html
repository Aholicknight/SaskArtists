Purpose:
    A design paradigm that unites data and processes into atomic building blocks.
Back to top

Concepts

    Operation - is what an object can do. Operations apply to an object and its data.

    Data Elements - fragments of information (data) that define an object or its behaviour. An object must operate on its own data.

    Encapsulation - the conceptual enclosing or containment of an object which ensures its independance (data must be operated on itself) and the moduality of objects in general.

    Class - a grouping of objects by data elements and operations. The grouping of objects provides for the application of object oriented concepts.

    Inheritance - when classes are arranged in hierarchy, the operations and data from a high class may apply to a lower class.

    Polymorphism - the modification of an inherited operation by a class or object.
