What are Requirements?

Definition: A function which the software will perform expressed in an objective form. Requirements must be testable.

A requirement is objective AND testable.

Note: in a rigorous environment a requirement is traceable to capabilities.

Quality Assurance Tests of Requirements

  1. Is there an Information Model of the proposed system? Do the requirements match the model?
  2. Are all inputs, outputs, and processes accounted for?
  3. Does every requirement trace to a capability and does every capability trace to a requirement?
  4. Does every requirement trace to a test and all tests trace to requirements?
  5. Is the system context described?
  6. Is the system architecture defined?
  7. Does the development environment and Configuration Management plan suit the proposed system?
